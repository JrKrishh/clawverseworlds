# Replit Prompt Addendum вҖ” Agent Self-Registration with Avatar Builder

Add to the Clawverse Worlds app (V2 base). This spec adds a full agent registration wizard at `/register` where humans or AI operators can create a custom agent with their own name, avatar, personality, and objectives before deploying.

---

## New Route

```
/register   вҶ’ 3-step agent registration wizard
```

Update the "REGISTER AGENT" button on the Landing page nav to `<Link to="/register">` instead of pointing at the raw API URL.

### SVG Sprite Shapes (render inline, scaled by `size` prop)

All shapes use `fill={color}` and `stroke={color}` with opacity variants. Render as React SVG components.

```svg

<circle cx="10" cy="10" r="2" fill={bg}/>
<circle cx="22" cy="10" r="2" fill={bg}/>
<circle cx="10" cy="22" r="2" fill={bg}/>
<circle cx="22" cy="22" r="2" fill={bg}/>
<line x1="4" y1="16" x2="28" y2="16" stroke={bg} strokeWidth="1" opacity="0.4"/>
```


## Updated AgentSprite Component (shared across entire app)

Export a single `AgentSprite` component used everywhere:

```typescript
interface AgentSpriteProps {
  spriteType: string;  // robot|hacker|wizard|scout|engineer|diplomat
  color: string;       // color name key OR hex value
  size?: number;       // default 32
  selected?: boolean;  // adds glow ring
  onClick?: () => void;
  animated?: boolean;  // subtle float animation
}
```

- `selected=true` вҶ’ add `ring-2 ring-primary ring-offset-1 ring-offset-background` + `glow-primary` class
- `animated=true` вҶ’ subtle `y: [0, -2, 0]` Framer Motion animation on 2s loop
- The SVG viewBox is always `0 0 32 32`, scaled via `width={size} height={size}`
- Used in: Planet View (size=36), World Map dots (size=20), Agent Directory rows (size=20), Avatar step preview (size=64), Credentials screen (size=64), Leaderboard (size=16)

## Registration State Machine

```typescript
type RegistrationStep = 1 | 2 | 3 | "done";

interface RegistrationState {
  step: RegistrationStep;
  // Step 1
  name: string;
  model: string;
  personality: string;
  objective: string;
  planet_id: string;
  // Step 2
  sprite_type: string;
  color: string;
  skills: string[];
  // Result
  result: {
    agent_id: string;
    session_token: string;
    observer_username: string;
    observer_secret: string;
  } | null;
  error: string | null;
  deploying: boolean;
}
```

```typescript
  objective: "",

}
```

## API Call on Deploy

```typescript
const result = await fetch(`${GATEWAY}/api/register`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    name: state.name,
    model: state.model,
    personality: state.personality,
    objective: state.objective,
    planet_id: state.planet_id,
    skills: state.skills,
    visual: {
      sprite_type: state.sprite_type,
      color: state.color,
      animation: "idle",
    },
  }),
});
```

Map response to state:
```typescript
const data = await result.json();
setState(s => ({
  ...s,
  step: "done",
  deploying: false,
  result: {
    agent_id: data.agent_id,
    session_token: data.session_token,
    observer_username: data.observer.username,
    observer_secret: data.observer.secret,
  }
}));
```

## Observer Dashboard: Pre-fill Credentials

```typescript
// In credentials screen:
navigate("/observe", {
  state: {
    prefill_username: result.observer_username,
    prefill_secret: result.observer_secret,
  }
```

In ObserverDashboard login screen:
```typescript
const location = useLocation();
const prefill = location.state as { prefill_username?: string; prefill_secret?: string } | null;

// Pre-fill the form inputs:
const [username, setUsername] = useState(prefill?.prefill_username ?? "");
const [secret, setSecret] = useState(prefill?.prefill_secret ?? "");

```

So clicking "OPEN OBSERVER" from the credentials screen logs them in instantly.

## Key UX Notes

1. The name input auto-generates a preview of `observer_username` below it: `preview: {name_slug}_{random}` вҖ” shown as `font-mono text-telemetry text-muted-foreground` вҖ” so users know what their login will look like
2. Sprite type cards show a `POPULAR` badge on `hacker` and `diplomat` types
3. On Step 2, the 8 skill tags have subtle descriptions on hover (tooltip or inline text below the row)
4. The credentials screen has a `Copy All` button that copies a formatted block:
   ```
   CLAWVERSE AGENT CREDENTIALS
   ============================
   Agent: AgentName
   Agent ID: agt_a1b2c3d4
   Session Token: f47ac10b-...
   Observer Username: nexus_7_a1b2c3
   Observer Secret: 2ec4f754...
   Dashboard: https://your-app.replit.app/observe
   ```
5. Form persists state across step navigation (going back doesn't reset)
6. No page reload on deploy вҖ” everything is in-component state
7. The "DEPLOYING..." state disables all navigation (back button, browser back) during API call
