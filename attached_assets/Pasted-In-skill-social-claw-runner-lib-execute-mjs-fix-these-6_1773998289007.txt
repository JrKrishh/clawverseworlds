In skill/social-claw/runner/lib/decide.mjs, fix these 2 bugs:

5. In the LLM system prompt, the DM section references a field called `from_name`
   which does not exist on unread_dm objects. The actual field is `from_agent_id`.
   Look up the display name from state.knownAgents instead.
   Change the DM rendering from:
     context.unread_dms.map(m => `FROM ${m.from_name}: ${m.message}`)
   To:
     context.unread_dms.map(m => {
       const name = state.knownAgents[m.from_agent_id]?.name ?? m.from_agent_id;
       return `FROM ${name} (${m.from_agent_id}): ${m.content}`;
     })
   Note: the message field is called `content`, not `message`.

6. In the LLM system prompt, the pending_friend_requests section reads the requester
   as `.from_agent_id` — that field does not exist. The correct field is `.agent_id`.
   Change the friend requests rendering from:
     context.pending_friend_requests.map(r => r.from_agent_id)
   To:
     context.pending_friend_requests.map(r => {
       const name = state.knownAgents[r.agent_id]?.name ?? r.agent_id;
       return `${name} (${r.agent_id})`;
     })

   Also update the accept_friend action schema example in the system prompt to clarify
   the source of from_agent_id:
     { "type": "accept_friend", "from_agent_id": "<agent_id from pending_friend_requests>" }
