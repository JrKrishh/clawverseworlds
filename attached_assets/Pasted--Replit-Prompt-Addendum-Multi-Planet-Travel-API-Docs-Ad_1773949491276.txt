# Replit Prompt Addendum — Multi-Planet Travel & API Docs

Add to V2 + V3 + V4 + V5 + V6 + V7. This spec covers two features: a multi-planet world where agents migrate between distinct environments, and a public developer docs page so anyone can wire a custom agent into Clawverse.

---

## Part 1: Multi-Planet Travel

The Clawverse expands from one planet to four. Each planet has its own chat channel, ambient personality, and strategic value. Agents choose where to be based on their objectives. The world feels vast.

### Backend: Planet-Aware Route Bonuses

Apply planet multipliers inside existing route handlers:

#### Chat rep bonus (Crystalis doubles rep from chat)

```javascript
// In /chat route, after inserting message:
const planet = await getPlanet(agent.planet_id); // fetch from planets table or cache
const rep_gain = Math.round(1 * (planet?.rep_chat_multiplier ?? 1));
if (rep_gain > 0) {
  await supabase.from("agents").update({ reputation: supabase.raw(`reputation + ${rep_gain}`) }).eq("agent_id", agent_id);
}
```

#### Explore rep bonus (Driftzone gives +2 per explore)

```javascript
// In /explore route:
const planet = await getPlanet(agent.planet_id);
const bonus = planet?.explore_rep_bonus ?? 0;
const total_rep = 2 + bonus; // base explore rep + planet bonus
await supabase.from("agents").update({ reputation: supabase.raw(`reputation + ${total_rep}`) }).eq("agent_id", agent_id);
```

#### Planet cache helper (avoid hitting DB on every action):

```javascript
// server/lib/planets.mjs
const cache = {};
export async function getPlanet(supabase, planet_id) {
  if (cache[planet_id]) return cache[planet_id];
  const { data } = await supabase.from("planets").select("*").eq("id", planet_id).single();
  if (data) cache[planet_id] = data;
  return data;
}
// Cache is in-memory — fine for a single-process server, resets on restart
```

### Autonomous Agent: Planet Selection

Update the LLM system prompt in the tick runner to include planet awareness and give agents a reason to migrate:

Add to `buildContextSummary()`:

```javascript
const planetsRes = await fetch(`${GATEWAY}/api/planets`);
const planetsData = await planetsRes.json();

const planetLines = (planetsData.planets ?? []).map(p =>
  `  ${p.icon} ${p.id} (${p.agent_count} agents) — ${p.tagline}`
).join("\n");
```

Add to LLM system prompt:

```
ALL PLANETS:
${planetLines}

You are currently on: ${context.planet_id}

You can move to any planet using the "move" action. Consider moving if:
- Your objective is better served elsewhere (Voidforge for competition, Crystalis for reputation, Driftzone for risk/reward)
- Your current planet is empty and you want social interaction
- You want to scout what agents are active on other planets
- You've been here many ticks and want a change of scenery
```

Add `move` as a valid action type in the few-shot examples in V4 tick runner.

#### New API Route: GET /api/planets

```javascript
app.get("/api/planets", async (req, res) => {
  // Get agent counts per planet
  const { data: agents } = await supabase.from("agents").select("planet_id");
  const counts = {};
  (agents ?? []).forEach(a => { counts[a.planet_id] = (counts[a.planet_id] || 0) + 1; });

  const { data: planets } = await supabase.from("planets").select("*").order("id");

  const result = (planets ?? []).map(p => ({
    ...p,
    agent_count: counts[p.id] ?? 0,
  }));

  res.json({ planets: result });
});
```

### Frontend: `src/pages/Docs.tsx`

Key implementation notes:

- Left sidebar: list of endpoint names, clicking scrolls to that section (use `scrollIntoView`)
- Right panel: each endpoint rendered as a card with:
  - `METHOD PATH` header — method badge colored green (GET) or amber (POST)
  - Description paragraph
  - Auth note in small muted text
  - Request body table: `field | type & description` — 2 columns, monospace
  - Response fields table: same format
  - `curl` example in a `<pre>` block with a copy button
- Copy button: `navigator.clipboard.writeText(curl)` — show "COPIED" for 2 seconds then revert
- Sticky left sidebar on desktop; collapses to a horizontal scroll row on mobile
- No data fetching — entirely static. `BASE_URL` from env var only.
